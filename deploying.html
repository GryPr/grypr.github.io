Deploying a Go REST API on Docker in a Ubuntu VM

Kevin Prom

These past few months, I've been learning Go to use it in developping backend servers for projects I may have in the future. This article doesn't necessarily go over the actual Go code, but will mostly go over how I deployed an existing application to Docker in a Ubuntu 20.0.4 LTS virtual machine for testing purposes.

Pre-Requisites

These are the tools that I've used in this article.

  • Oracle Virtualbox (v6.1.6)
  • Ubuntu VM (20.0.4 LTS)
  • Docker installed on the Ubuntu VM
    • I've simply installed Docker during the Ubuntu installation as shown on the right

Project Specifications

The Go program is a basic "to-do" app REST API that implements all CRUD operations onto a Postgresql database.

The Go code is available here.

The important things to take into consideration are:

  • The connection to the Postgresql database
    • The Postgresql database is hosted on the host machine
  • The exposed REST API address
    • The API should be accessible from the host machine

Shared Folder

The first hurdle we're coming across is how to pass files to the Ubuntu VM. There are many ways to do so, but I've opted to use shared folders due to how easily you can edit files on your host and have the changes appear immediately in the VM.

  1. In your VirtualBox Manager, access your VM's settings menu
  2. Navigate to the Shared Folders tab
  3. On the right sidebar, click on the Add Shared Folder button
  4. Input your folder's path and check Auto-Mount and Make Permanent
  5. Install the VirtualBox Guest Utils
    sudo apt-get install virtualbox-guest-utils
  6. Make a guest folder directory
    $ mkdir guestFolder
  7. Mount your host folder to the guest folder created previously
    sudo mount -t vboxsf hostFolder guestFolder
